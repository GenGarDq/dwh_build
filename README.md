В данном случае cделаем допущение, что факты пришли в .csv файлах, а план - как .xslx.
В качестве СУБД будем использовать postgreSQL (транзакционная, легко развертывается), как ETL-инструмент - Apache Airflow с local executor.
Для локального выполнения задания развернем эти сервисы через Docker.
Данные в фактах и планах уже достаточно неплохого качества (нет дублей строк, нет null-значений, выбросов тоже не найдено)

Преобразования в фактах:
1. Дату преобразуем к формату yyyy-mm-dd, дату с временем yyyy-mm-dd hh:mm
2. Час открытия и час закрытия - это лишние, избыточные данные. При необходимости их можно легко вытащить из времени открытия и закрытия. Эти колонки убираем.
3. Блюдо, категорию блюда и тип заказа выносим в отдельные справочники, которые будут связаны с исходной таблицей через id.
4. Типы блюд с буквой Д. и без считаем за разные, в доставке и в самом ресторане лучше разделять меню по бизнес логике.
5. Все данные приводим к единому регистру (нижнему в данном случае).
6. Приводим типы данных к необходимым для вставки в БД.
7. Есть строки с лишним переносом данных, но они корректно парсятся при указании разделителя ',' (поэтому, кстати, в таблице postgres будет меньше строк, чем в источнике)
8. Если в записях с одинаковым номером чека и временем закрытия итоговая стоимость равна нулю - скорее всего это тестовые\ошибочные записи. Их можно удалить.
9. Есть записи блюд без добавок и с добавками (+) - одно и то же, т.к. добавка прописывается отдельно. Убираем лишние плюсы.

Преобразования в планах:
1. Приведение колонок с подзаголовками к обычной БД структуре без подколонок
2. Удаляем "Итого". Если оно нужно для постоянного доступа - можно сделать представление. Или по ad-hoc запросу.
3. Таблица планов идентична и в core и в ods слоях. Из-за маленького размера можно продублировать, либо же оставить только в core. В данный момент продублировано.

Структура DWH:
1. STG - наши сырые данные, в папке files
2. ODS - очищенные стандартизированные данные
3. Core - нормализованные и связанные данные
4. Marts - слой витрин, в данном задании не используется, в реальном проекте обязательно есть
